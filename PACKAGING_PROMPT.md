# 📦 נוסח לאריזת פרויקט לתבנית לשיתוף

**למי זה מיועד:** מישהי שבנתה משהו (דשבורד, אוטומציה, כלי) ורוצה לארוז אותו לתבנית נקייה
שחברה אחרת תוכל לקחת ולהטמיע אצלה — בלי הפרטים והסודות האישיים.

**איך משתמשים:** פותחים Claude Code **בתוך תיקיית הפרויקט הקיים** ומדביקים את הבלוק שבין הקווים.

---

אני רוצה לארוז את הפרויקט הזה לתבנית נקייה לשיתוף, כדי שחברה תוכל לקחת ולהטמיע אצלה מאפס
בלי שום פרט אישי שלי. עשה את כל העבודה, ותעצור לשאול רק כשחייבים אותי. בצע כך:

**1. הבן מה נבנה כאן**
- סרוק את הפרויקט והכן סיכום קצר: מה זה עושה, מאיזה חלקים הוא מורכב (שרת/דשבורד/DB/אינטגרציות),
  ואילו שירותים חיצוניים הוא צריך (Railway/Netlify/Vercel, מסד נתונים, Google, מייל, API-ים).

**2. צור עותק תבנית נפרד**
- צור תיקייה חדשה נפרדת (למשל `<project>-template`) והעתק אליה את קוד המקור בלבד —
  **בלי** `node_modules`, `.git`, קבצי סביבה, אישורים, או קבצי נתונים/session.

**3. נקה כל פרט אישי וסוד — קריטי**
- עבור על כל הקוד וחלץ החוצה למשתני סביבה כל: מפתחות API, טוקנים, סיסמאות, connection strings,
  כתובות מייל, מספרי טלפון, מזהי קבוצות/גיליונות/פרויקטים, כתובות דומיין אישיות, ושמות פרטיים.
- החלף ערכים אישיים מקודדים-קשיח בקריאה מ-`process.env.XXX` (או המקבילה בשפה), עם ברירת מחדל ריקה.
- הרץ חיפוש אחרון לווידוא ניקיון (מיילים, טלפונים בפורמט 972/05, מפתחות ארוכים, שמות אישיים) —
  ואל תשאיר שום דבר אמיתי בקוד.
- אם יש נתונים לדוגמה (רשומות אמיתיות, לקוחות) — החלף בדוגמאות גנריות או רוקן.

**4. הוסף קבצי הקמה**
- `.gitignore` שמתעלם מ-node_modules, קבצי סביבה, אישורים וקבצי נתונים.
- `.env.example` עם **כל** משתני הסביבה הנדרשים, הערות מה כל אחד, ומה חובה מול אופציונלי.
- `README.md` בעברית: מה הכלי עושה, דרישות, והוראות הקמה.
- אם הפרויקט כולל דשבורד/ממשק — צלם או תאר במילים איך הוא נראה ומה הוא מציג.

**5. תעד תקלות ולקחים**
- הוסף ל-README קטע "תקלות נפוצות — כבר פתורות" עם כל באג/מכשול שנתקלנו בו ואיך נפתר,
  כדי שמי שמטמיע לא יבזבז על זה זמן.

**6. כתוב נוסח התקנה מוכן (SETUP_PROMPT.md)**
- כתוב נוסח בעברית שמישהי אחרת תוכל להדביק ב-Claude Code שלה, והוא יקים לה את הכל מאפס:
  שכפול מ-GitHub, התקנת תלויות, פריסה לשירות הרלוונטי, הקמת מסד נתונים אם צריך, והגדרת משתני הסביבה.
- הנוסח צריך: (א) לפתוח בהסבר מה יידרש ממנה עצמה (התחברות לחשבונות, תשלום אם יש, חיבורים ידניים);
  (ב) לאסוף ממנה מראש את הפרטים להתאמה אישית; (ג) לעצור רק בנקודות שחייבות אותה.

**7. פרסם ל-GitHub**
- ודא שאני מחוברת (`gh auth status`), צור repo חדש בשם `<project>-template` ודחוף.
- שאל אותי אם לפרסם ציבורי (Public — קל לשתף) או פרטי (Private — צריך להוסיף כל אחת ידנית).
- בסוף תן לי את קישור ה-repo ואת הקישור הישיר ל-SETUP_PROMPT.md, מוכנים לשליחה.

חשוב מאוד: אל תדחוף שום סוד או פרט אישי ל-GitHub. אם אתה לא בטוח אם ערך מסוים רגיש — שאל אותי.
בסיום תן לי סיכום קצר: מה נכלל בתבנית, אילו משתני סביבה צריך למלא, ומה נשאר למי שמטמיעה לעשות.
