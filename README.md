# WhatsApp Scheduler — תבנית מוכנה

מערכת לשליחת הודעות וואטסאפ **מתוזמנות** לקבוצות (טקסט או תמונה), עם ממשק ניהול בעברית.
נבנתה עם Baileys + PostgreSQL, רצה על Railway.

## מה זה נותן
- 📅 תזמון הודעות לקבוצות לפי תאריך ושעה (שעון ישראל)
- 🖼️ טקסט או תמונה עם כיתוב
- 📁 ארגון קבוצות בקטגוריות בטופס הבחירה
- 💾 שמירה ב-PostgreSQL — שורד restarts ו-redeploys בלי לסרוק QR מחדש
- 🔔 התראה בוואטסאפ/מייל **רק** כשהבוט מתנתק
- 🛡️ הגנה מפני שליחה כפולה

## התקנה מהירה
פתחו את [SETUP_PROMPT.md](SETUP_PROMPT.md), העתיקו את הנוסח והדביקו ב-Claude Code — הוא בונה הכל.
או ידנית: ראו את השלבים ב-SETUP_PROMPT.md.

## משתני סביבה
ראו [.env.example](.env.example). היחיד שחובה הוא `DATABASE_URL` (מוזרק אוטומטית ע"י Postgres ב-Railway).

## שימוש יומיומי
- **עמוד ראשי** — יצירת הודעה מתוזמנת + צפייה בממתינות/שנשלחו
- **/groups** — הוספת קבוצות שהבוט חבר בהן לרשימת היעדים
- **/qr** — סריקת QR לחיבור/חיבור מחדש
- **/state** — בדיקת סטטוס (connected / qr / disconnected)

## ⚠️ תקלות נפוצות — כבר פתורות בתבנית (חוסך שעות)
כל הפתרונות האלה כבר מובנים בקוד. חשוב להכיר כדי לא ליצור אותם מחדש:

1. **שליחה כפולה — הכי חשוב:** אם רצים **שני שירותים / רפליקות** של אותו repo (למשל שירות שני ב-Railway, או replicas>1), *שניהם* מתחברים לאותו מסד נתונים ושולחים כל הודעה פעמיים.
   👉 הקפידו על **שירות אחד בלבד** ו-**replica אחת**. אם רואים כפילות — קודם כל לבדוק כמה שירותים רצים בפרויקט.
2. **שליחה כפולה #2:** מרוץ בין הרצות חופפות של המתזמן. פתור בתבנית ע"י מנעול `schedulerRunning` שמדלג על tick חדש כל עוד הקודם רץ.
3. **תקוע על "ממתין ל-QR" אחרי התנתקות:** פותחים `<APP_URL>/reset-session?confirm=yes` — מנקה creds ישנים, שומר את ההודעות, ומפיק QR חדש. הקוד גם מרפא את עצמו אוטומטית בכל logout.
4. **QR מבקש סריקה מחדש בכל restart:** נפתר ע"י שמירת ה-session ב-PostgreSQL (ולא בזיכרון). לכן **חובה** להוסיף Postgres לפרויקט — בלעדיו כל דפלוי מנתק.
5. **קבוצה שהוספת לקוד לא מופיעה:** `loadGroups()` ממזג את הקבוצות השמורות עם רשימת ברירת המחדל שבקוד (ולא דורס). מחיקות נשמרות בנפרד.
6. **הודעות "מחובר ופועל" מציקות:** הוסרו — התבנית שולחת התראה **רק** בהתנתקות.
7. **"ממתין להודעה זו" אצל הנמענים:** מגבלה של Baileys. התבנית מצמצמת זאת עם חימום-הצפנה שקט לפני שליחה + שהיות. עוזר הכי הרבה כשהטלפון הראשי מחובר לאינטרנט בזמן השליחה.

## הערה
Baileys הוא ממשק לא-רשמי לוואטסאפ. שליחה מרובה לקבוצות עלולה לעיתים לגרום לוואטסאפ לנתק את המכשיר. השתמשו באחריות ובקצב סביר.
